# 用户注册登录功能说明

## 功能概述

本项目已实现完整的用户注册和登录功能，包括账户管理、用户信息展示、状态持久化等核心功能。

## 核心概念区分

### 账户 vs 显示名称

| 概念 | 说明 | 用途 | 示例 | 修改频率 |
|------|------|------|------|----------|
| **账户** | 用户用于登录的唯一标识 | 身份验证、后台系统 | `user@example.com`、`john123` | 低（一般不可修改） |
| **显示名称** | 用户在界面中展示的名称 | 用户交流、界面展示 | `张三`、`李四`、`John Doe` | 高（可随时修改） |

**重要说明**：
- 账户是后台系统使用的唯一标识，用于登录和身份验证
- 显示名称是用户用于交流的称呼，其他用户通过这个名称识别用户
- 两个概念完全独立，修改显示名称不影响账户登录功能

## 功能模块

### 1. 用户注册

**页面路由**：`/register`

**功能特性**：
- 账户输入（至少3个字符）
- 显示名称输入（昵称）
- 密码设置（至少6个字符）
- 密码确认（验证一致性）
- 头像上传（可选，支持图片预览）
- 个人简介（可选，最多200字）
- 账户唯一性验证
- 注册成功后自动跳转到登录页

**表单验证**：
- ✅ 必填字段验证（账户、显示名称、密码）
- ✅ 账户长度验证（至少3个字符）
- ✅ 密码长度验证（至少6个字符）
- ✅ 密码一致性验证
- ✅ 账户唯一性验证（防止重复注册）
- ✅ 个人简介字数限制（200字）

**文件位置**：`src/views/RegisterView.vue`

---

### 2. 用户登录

**页面路由**：`/login`

**功能特性**：
- 账户输入
- 密码输入
- 表单验证（必填检查）
- 错误提示
- 登录成功后跳转到首页
- 支持跳转到注册页面

**登录流程**：
1. 用户输入账户和密码
2. 点击登录按钮
3. 系统验证账户和密码
4. 验证成功：跳转到首页，显示用户信息
5. 验证失败：显示错误提示信息

**文件位置**：`src/views/LoginView.vue`

---

### 3. 用户信息展示

**页面路由**：`/`

**功能特性**：
- 根据登录状态显示不同内容
- 未登录：显示欢迎信息和登录按钮
- 已登录：显示用户详细信息
  - 头像（如果已上传）
  - 显示名称（昵称）
  - 账户信息
  - 个人简介（如果已填写）
- 支持退出登录

**文件位置**：`src/views/HomeView.vue`

---

### 4. 导航栏

**功能特性**：
- 根据登录状态动态显示导航项
- 未登录：显示"登录"和"注册"链接
- 已登录：显示用户显示名称和"退出"按钮
- 始终显示"首页"和"关于"链接

**文件位置**：`src/App.vue`

## 状态管理

### UserStore (`src/stores/user.ts`)

#### 数据接口

```typescript
interface User {
  id: string          // 用户唯一标识符
  account: string     // 登录账户
  displayName: string // 显示名称（昵称）
  avatar?: string     // 用户头像 URL（可选）
  bio?: string        // 个人简介（可选）
}

interface RegisterData {
  account: string     // 登录账户
  displayName: string // 显示名称（昵称）
  password: string    // 登录密码
  avatar?: string     // 头像图片（可选）
  bio?: string        // 个人简介（可选）
}
```

#### 状态

| 状态 | 类型 | 说明 |
|------|------|------|
| `user` | `User \| null` | 当前登录用户信息 |
| `users` | `User[]` | 已注册用户列表 |
| `isLoggedIn` | `Computed<boolean>` | 登录状态（true = 已登录） |

#### 方法

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `login(account, password)` | `account: string`, `password: string` | `boolean` | 用户登录，成功返回 true |
| `logout()` | - | `void` | 用户退出登录 |
| `register(userData)` | `userData: RegisterData` | `boolean` | 用户注册，成功返回 true |

#### 使用示例

```typescript
import { useUserStore } from '@/stores/user'

const userStore = useUserStore()

// 登录
const success = userStore.login('user@example.com', 'password123')

// 退出登录
userStore.logout()

// 注册
const registerSuccess = userStore.register({
  account: 'user@example.com',
  displayName: '张三',
  password: 'password123',
  bio: '这是我的个人简介'
})

// 获取当前用户
const currentUser = userStore.user

// 检查登录状态
if (userStore.isLoggedIn) {
  console.log('用户已登录')
}
```

## 数据存储说明

### 当前实现

- **存储方式**：内存存储（Pinia）
- **数据持久化**：无（刷新页面后数据重置）
- **适用场景**：开发、演示、原型

### 生产环境建议

在实际生产环境中，建议使用以下方式持久化用户数据：

1. **前端存储**：
   - localStorage：存储用户基本信息
   - sessionStorage：存储会话信息
   - IndexedDB：存储大量数据

2. **后端存储**：
   - RESTful API：与后端服务器交互
   - GraphQL：灵活的数据查询
   - JWT Token：用户身份验证

3. **安全建议**：
   - 密码加密存储（bcrypt、argon2）
   - 使用 HTTPS 传输
   - 实现 CSRF 防护
   - 添加 Token 过期机制

## 注意事项

### 开发注意事项

1. **数据持久化**
   - 当前数据存储在内存中
   - 刷新页面后所有数据会重置
   - 生产环境需要实现持久化存储

2. **密码安全**
   - 当前密码仅验证非空
   - 生产环境应使用加密存储
   - 建议使用 bcrypt 或 argon2

3. **头像存储**
   - 当前使用 Base64 编码存储
   - 仅适合演示使用
   - 生产环境应上传到服务器或对象存储

4. **表单验证**
   - 当前验证在客户端进行
   - 生产环境应同时进行服务端验证
   - 防止绕过前端验证
